# Монетизация TasteCompass — зафиксировано (v1)

Документ для возврата к теме позже. Не план действий по коду — только рамка и гипотезы.

## Кто обычно платит в таких продуктах

| Плательщик | За что | Заметка |
|------------|--------|---------|
| Заведения / ивенты | Показ в подборке, лиды, бронь, «сегодня вечером» | Прямая связь с ценностью; нужны охват и доверие |
| Афиша / билеты | Партнёрка, CPA за покупку | Масштаб; интеграции и юрисдикция |
| Сети кинотеатров / дистрибьюторы | Спонсорские слоты, аналитика спроса | Крупные бюджеты; долгие циклы, метрики |
| Бренды (напитки, транспорт, lifestyle) | Спонсорские подборки, нативно | Риск размыть доверие «компаса вкуса» |
| Город / туризм / карты-приложения | White-label, API, лицензия | B2B, стабильнее; нужен продукт «под ключ» |
| Пользователь (подписка) | Премиум без рекламы, поездки, больше контекста | На ранней стадии сложно; позже — ниша «частые выходы» |

**Реклама баннерами** — быстрый доход, но для доверительного продукта часто вредит бренду; предпочтительнее **честно помеченные** партнёрские форматы.

## Рабочая гипотеза направления

1. **Один основной плательщик-гипотеза** на горизонте 3–6 месяцев (например: заведения + CPA/бронь **или** афиша + партнёрка), с измеримой воронкой: показ → клик → действие.
2. **Не распыляться** на «кино + бары + бренды» до появления **одного** сценария, где продукт сильнее альтернатив (например: «вечер в городе X» для приезжих).
3. **Деньги ближе к транзакции**: билет, стол, бронь — тогда модель «мы привели гостя» считается.
