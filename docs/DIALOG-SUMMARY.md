# Саммари диалога (контекст)

**Дата:** 2025-03-22  
**Проект:** RECOMMEND AI

## Что обсудили

1. **Почему Facebook выжил, а Myspace/Foursquare — нет:** сетевой эффект, копирование фич, реклама/лента, поглощения (Instagram, WhatsApp), реальные имена, мобильный поворот, привычка как инфраструктура.
2. **Социальный граф** — карта связей (узлы/рёбра); чем плотнее, тем дороже уход.
3. **Идея продукта:** рекомендации и люди по **вкусам** из разных сервисов (кино, музыка, события). Сильная форма — не «ещё одна соцсеть», а **инструмент / taste engine** (метафора компаса, не паспорта).
4. **Рабочее имя:** **TasteCompass** (ориентация в культуре; без ассоциаций с «госпаспортом»). Альтернатива TastePassport отклонена из-за ассоциаций.
5. **Конкуренты/аналоги:** Taste.io (фильмы), Tastebuds, Letterboxd, Music-Map, Gnoosic и др. — обычно один слой; глобального объединённого taste graph почти нет (данные, фрагментация, приватность, экономика платформ, контекстность вкусов).
6. **Барьеры matching по вкусу:** разреженность данных, иерархия сцен, контекст (режимы), холодный старт, социальный смысл «совпадения», малая плотность нишевых сцен в городе. Выход — **taste space / кластеры сцен**, не сырые пересечения списков.
7. **Архитектура движка:** входы (интеграции + опрос + поведение), нормализация сущностей, вектор вкуса, кластеры, retrieval + ранжирование, мульти-режим, объяснимость, петля обучения; почему мало кто довёл до конца (данные, стоимость качества, курица-яйцо по городам, T&S, ожидания).
8. **Два варианта запуска:** (1) культурный навигатор для путешествий — места/ивенты, люди вторично; (2) сеть людей по вкусам — тяжелее из-за плотности. Рационально: сначала инструмент, потом социальный слой.
9. **Реестр документов продукта:** полный нумерованный список артефактов — в `PRODUCT-DOCUMENTATION-CHECKLIST.md`.
